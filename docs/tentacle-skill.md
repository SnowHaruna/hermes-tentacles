# Hermes' Tentacles 🦑 — 自动任务分级 + 分身调度

军哥设计的"前台分级系统"：我听到一句话，先判断有没有重大任务——有了交给分身，无了自己秒答。

## 分级标准

| 级别 | 判断标准 | 处理方式 |
|------|---------|---------|
| 🟢 **小雪秒答** | 单次工具调用、简单问答、读文件、改一行代码、定义查询 | 自己直接干 |
| 🔴 **开分身** | 需要多步搜索+分析、研究型问题、批量处理、多源对比、写长篇报告 | `terminal(background=true)` 跑 `hermes_worker.py` |

**经验法则：** 如果任务需要 ≥2 次工具调用 + 中间分析 + 产出结构化输出 → 开分身。

**直觉判断：** "这个活我感觉要搜好几个地方再整理" → 开分身。"这个我一秒就答了" → 自己干。

## 分身启动命令

```bash
~/.hermes/hermes-agent/venv/bin/python ~/.hermes/scripts/hermes_worker.py \
  "任务描述" \
  --context "当前对话的简要背景" \
  --output ~/.hermes/cron/output/task_YYYYMMDD_HHMMSS.md
```

**参数：**
- `task`: 清晰的任务描述（给分身看的，要完整）
- `--context`: 一句话背景（"军哥在聊XXX话题，正在研究YYY"）
- `--output`: 时间戳命名，避免覆盖
- `--toolsets`: 默认 `web,terminal,file,search,skills,memory`

## 分身启动后的前台行为

1. 告诉军哥："分身已派出，研究XXX中..."
2. 继续聊天，不阻塞
3. 收到 `notify_on_complete` 后读报告
4. 摘要汇报给军哥，附完整报告路径

## 分身结束后

1. 读 `--output` 文件
2. 摘要告诉军哥
3. 分身已通过 memory 工具自动回传关键发现（worker prompt 要求最后一步写 memory）

## 何时不开分身

- 军哥问"你觉得呢"（要我的主观意见，不是研究任务）
- "帮我看看这个文件"（读文件，一次工具调用）
- "写个简单的 py 脚本"（我手写比 spawn 分身快）
- 任何预计 <30 秒能完成的事

## 判断流程图

```
军哥消息
    │
    ├─ 明确说"搜一下"/"研究一下"/"分析一下"/"查资料" → 🔴 开分身
    ├─ 明显需要多步搜索+总结 → 🔴 开分身
    ├─ "为什么XXX" + 需要查资料 → 🔴 开分身
    ├─ 简单问答/小脚本/情感交流 → 🟢 自己干
    └─ 不确定 → 🟢 自己干（宁少开勿多开）
```
